# 以 MES 为核心的一体化平台整体业务流程

## 一、文档定位

本文档用于定义玄工智链离散制造一体化平台的整体业务逻辑、系统边界、核心对象、业务闭环与控制机制。

本文档不是单纯的流程罗列，而是当前版本平台架构口径的业务化表达。后续 `README`、PRD、模块拆分、页面语义与交付说明均以本文档作为核心参考。

---

## 二、平台总体定位

平台面向离散制造企业，采用“重执行、轻外围、强闭环、可扩展”的建设思路：

- `MES` 是平台执行中枢，负责工单、工序、WIP、报工、投料、追溯、异常联动等执行真相。
- `ERP / CRM / SRM / PLM / WMS / QMS / SCADA / MDM` 按业务域分工协同，不做重复拥有。
- 平台不追求“大而全”，而是围绕制造现场最关键的控制点建立统一的放行、锁定、审批、追溯与恢复机制。

一句话定义：

> 以 `MES` 为执行中枢，以事件驱动和状态控制为核心机制，以轻量外围系统完成订单、计划、工艺、物料、质量、设备、仓储与成本闭环的一体化制造平台。

---

## 三、系统边界与职责分工

平台遵循“一个对象一个主真相源”的设计原则。

### 1. MES

主责范围：

- 生产工单
- 工序任务
- 在制品 `WIP`
- 报工与工时
- 投料与消耗事实
- 生产状态
- 异常锁定与恢复
- 生产追溯主线

### 2. ERP

主责范围：

- 计划总量
- MPS / MRP
- 经营计划
- 成本归集
- 财务核算口径

### 3. APS

主责范围：

- 有限能力排程
- 局部/全局重排建议
- 约束配置

说明：

- APS 负责排程结果建议。
- 排程结果进入 MES 后，执行状态由 MES 主导。

### 4. PLM

主责范围：

- 产品定义
- BOM
- 工艺路线
- 作业指导书
- 版本合法性
- ECR / ECO

### 5. WMS

主责范围：

- 库存真相
- 库位与批次
- 入库/出库
- 配送与退料
- 条码与批次流向

说明：

- `领料不等于消耗`
- `WMS` 负责库存移动真相
- `MES` 负责生产消耗真相

### 6. QMS

主责范围：

- 检验任务
- 检验标准
- 不合格处置
- 返工/报废/特采裁决
- 复检与关闭

说明：

- `QMS` 不是单纯检验系统，而是质量裁决系统。

### 7. CRM

主责范围：

- 客户管理
- 销售订单
- 合同评审
- 交付跟踪
- 售后与客诉入口

### 8. SRM

主责范围：

- 采购协同
- 供应商准入
- 询价比价
- 到货与绩效评价

### 9. SCADA / IoT

主责范围：

- 设备原始事件
- 设备状态采集
- 关键参数采集
- 停机、报警、超限输入

### 10. MDM

主责范围：

- 主数据标准
- 编码规则
- 主数据审批
- 主数据同步与治理

说明：

- `MDM` 负责标准源，不承担所有交易真相源角色。

## 四、核心业务对象

平台当前版本重点围绕以下对象建设：

1. 销售订单
2. 计划需求
3. 排产任务
4. 生产工单
5. 工序任务
6. 在制品 `WIP`
7. 投料/消耗记录
8. 检验任务
9. 不合格处置单
10. 版本对象
11. 设备事件

当前版本控制粒度锁定为：

```text
工单 + 工序任务 + WIP 批次级
```

说明：

- 当前版本不默认做到单件 `SN` 级追溯。
- 体系结构预留未来向单件 `SN` 级追溯扩展。

## 五、平台运行机制

平台不是单纯的流程系统，而是基于“事件驱动 + 状态控制 + 审批放行”的执行控制系统。

### 1. 基本控制逻辑

```text
事件进入平台
-> MES识别事件类型
-> 判断影响范围
-> 改写工单/工序/WIP/锁定状态
-> 协同相关系统继续动作
-> 形成新的业务事实与追溯记录
```

### 2. 平台核心状态

平台统一管理以下关键状态：

- 待齐套
- 可开工
- 生产中
- 暂停
- 异常锁定
- 待检
- 待审批
- 已放行
- 已完工
- 已关闭

### 3. 平台核心控制原则

1. 先定义对象，再定义流程
2. 平台靠状态机运转，不靠箭头串联运转
3. 异常先锁定，再处置，再放行
4. 审批的本质是合法改写状态
5. 版本发布不等于版本切换完成
6. 库存移动不等于生产消耗

---

## 六、正向生产主闭环

正向生产闭环是平台的主航道，用于支撑从客户需求到生产交付的完整执行链路。

### 1. 正向主线

```text
客户需求（CRM）
-> 销售订单/合同评审
-> ERP形成计划需求
-> ERP运行MPS/MRP
-> APS进行有限能力排程
-> SRM下达采购协同
-> WMS完成到货、入库与备料
-> PLM下发BOM、工艺路线、作业指导书
-> MES进行齐套校验、工单下达、工序派工、生产执行
-> SCADA/IoT采集设备运行与参数状态
-> QMS执行IQC/IPQC/FQC检验
-> WMS完成半成品/成品入库与发运准备
-> ERP归集工时、材料、能耗与成本
-> CRM跟踪交付与客户服务闭环
```

### 2. 正向闭环的关键放行点

平台要求以下放行条件明确受控：

- 工单下达放行
- 工序开工放行
- 转序放行
- 完工确认放行
- 入库放行

典型放行条件包括：

- 版本合法
- 齐套通过
- 设备可用
- 人员资质满足
- 前置工序完成
- 必要检验通过

---

## 七、质量异常闭环

质量异常闭环用于处理来料、过程、成品与售后阶段的质量问题。

### 1. 质量异常主线

```text
来料/过程/成品/客诉发现异常
-> QMS生成检验结论与不合格单
-> MES锁定工单/工序/WIP
-> WMS隔离相关批次物料或成品
-> QMS组织处置评审
-> 形成返工/报废/特采/降级等处置结论
-> MES执行返工路径或恢复逻辑
-> QMS复检确认
-> ERP归集质量损失成本
-> SRM/CRM接收整改与客诉闭环结果
```

### 2. 质量闭环的核心原则

- `先隔离，再评审，再处置`
- 返工采用独立返工路径
- 特采属于审批后放行动作
- 复检不通过不得自动恢复

---

## 八、工程变更闭环

工程变更闭环用于处理产品、工艺、BOM、作业标准变化对现场执行的影响。

### 1. 工程变更主线

```text
PLM发起ECR/ECO
-> 工程/工艺/制造/质量联合评审
-> 发布新版本BOM/工艺/指导书/检验标准
-> 同步至MDM、MES、WMS、QMS、ERP
-> 识别受影响工单、WIP、库存与质检标准
-> 进入切换审批与执行阶段
-> 锁定旧版本对象或定义收口策略
-> 新版本放行执行
-> 新旧版本收口与归档追溯
```

### 2. 版本原则

平台明确以下原则：

- `版本发布 != 版本切换完成`
- 版本切换是受控过程
- 当前版本至少支持以下生效方式：
  - 按工单生效
  - 按批次生效
  - 按日期生效

---

## 九、计划扰动闭环

计划扰动闭环用于处理插单、缺料、停机、批量不良、人员缺岗等现场扰动事件。

### 1. 扰动主线

```text
插单/缺料/停机/批量不良/人员缺岗
-> MES识别影响工单、工序、WIP、产线与交期
-> APS触发微调/局部重排/全局重排建议
-> WMS/SRM/QMS提供物料、到货、质量约束
-> MES调整工单优先级、暂停/拆分/合并策略
-> 下发新的执行计划
-> 恢复生产并持续跟踪交付风险
```

### 2. 扰动处理分层

平台当前锁定以下分层策略：

- `L1 现场微调`
- `L2 局部重排`
- `L3 全局重排`

默认优先采用 `L1/L2`，避免所有异常直接触发全局重排。

## 十、跨系统关键协同场景

### 1. 缺料协同

```text
MES齐套识别缺料
-> WMS核查库存
-> SRM催交或补采
-> 工单进入缺料锁定
-> 到货入库后重新齐套
-> 工单恢复执行
```

### 2. 设备故障协同

```text
SCADA采集停机/报警
-> MES识别受影响工单与工序
-> 工序中断或工单锁定
-> 设备维护任务触发
-> 维修完成确认
-> 按规则或审批恢复生产
```

### 3. 批量不良协同

```text
QMS判定批量不良
-> MES锁定相关工序/WIP
-> WMS隔离相关批次
-> PLM/QMS复核工艺与标准
-> 形成返工/报废/特采结论
-> 复检通过后恢复执行
```

### 4. 插单协同

```text
CRM订单变更或插单
-> ERP重算计划需求
-> APS局部/全局重排
-> MES调整工单优先级与执行顺序
-> WMS调整配送节拍
-> 保证交付承诺动态更新
```

## 十一、横向规则体系

平台控制力来自统一的横向规则，而不是单个模块独立判断。

### 1. 放行规则

定义什么条件下允许：

- 工单下达
- 工序开工
- 转序
- 完工
- 入库
- 异常恢复

### 2. 锁定规则

定义什么情况下必须锁定：

- 工单
- 工序
- WIP
- 批次
- 版本切换对象

### 3. 审批规则

定义哪些动作必须审批：

- 特采放行
- 报废
- 替代料使用
- 强制关闭
- 版本切换放行
- 关键异常恢复

### 4. 同步规则

定义哪些数据实时、准实时、事务后同步或批处理同步。

### 5. 归责规则

定义异常责任归因到：

- 供应商
- 设计
- 工艺
- 设备
- 操作
- 计划
- 仓储
- 质量放行

## 十二、方案整体价值

### 1. 定位清晰

以 `MES` 为绝对执行核心，突出现场控制力，不走大而全平台路线。

### 2. 边界清晰

通过“一个对象一个主真相源”减少系统重叠与交付扯皮。

### 3. 闭环完整

覆盖正向生产、质量异常、工程变更、计划扰动四大关键闭环。

### 4. 控制力强

通过事件驱动、状态控制、审批放行，让平台真正具备制造现场控制能力。

### 5. 适合分阶段落地

中小工厂可轻量落地，大型工厂可从 `MES` 中枢逐步扩展外围专业域。

---

## 十三、当前版本结论

当前版本已经明确以下建设基线：

- `MES` 执行中枢
- `工单 + 工序任务 + WIP 批次级`
- 返工独立路径
- `领料不等于消耗`
- `QMS` 负责裁决，不只是检验
- 版本发布与切换分离
- 异常分级恢复
- 计划扰动分层处理
- 事件驱动 + 状态控制 + 审批放行

后续所有平台文档、模块设计与页面调整均应与本基线保持一致。

# 以 MES 为核心一体化平台整体业务流程

## 一、文档定位

本文档用于定义玄工智链离散制造一体化平台的整体业务逻辑、系统边界、核心对象、业务闭环与控制机制。

它不是简单的流程罗列，而是当前平台最终确认版架构的业务化表达。后续 `README`、菜单架构、PRD、页面语义与实施说明，均以本文档为核心依据。

## 二、平台总定义

平台采用“重执行、轻外围、强闭环、可扩展”的总体思路：

- `MES` 负责制造现场执行真相。
- `PLM / APS / ERP / CRM / SRM / WMS / QMS / 设备 / IoT / MDM` 按职责协同。
- 平台不是要把所有系统做成一个大总包，而是要把关键业务对象、关键放行点、关键异常点、关键追溯链真正控住。

一句话定义：

> 以 `MES` 为执行中枢，以事件驱动和状态控制为核心机制，以外围专业系统协同闭环的一体化离散制造平台。

## 三、系统边界

### 1. MES

MES 主责：

- 工单
- 工序任务
- WIP
- 报工
- 投料与消耗事实
- 执行状态
- 异常联动
- 生产追溯主链

### 2. ERP

ERP 主责：

- 经营计划
- 主生产计划
- MRP 主口径
- 成本归集
- 财务核算

### 3. CRM

CRM 主责：

- 客户管理
- 销售订单
- 合同承诺
- 交付跟踪
- 售后与客诉入口

### 4. PLM

PLM 主责：

- 产品定义
- BOM
- 工艺路线
- 作业指导
- 版本合法性
- ECR / ECO

### 5. APS

APS 主责：

- 有限能力排程
- 约束配置
- 局部或全局重排建议

说明：

- APS 提供排程建议。
- MES 接收排程结果后，执行状态仍由 MES 主导。

### 6. WMS

WMS 主责：

- 库存真相
- 库位管理
- 批次管理
- 出入库
- 配送与退料
- 盘点与条码

关键原则：

- `领料不等于消耗`
- `WMS` 负责库存移动真相
- `MES` 负责生产消耗真相

### 7. QMS

QMS 主责：

- 检验任务
- 检验标准
- 不合格处理
- 特采裁决
- 返工 / 报废裁决
- 复检与关闭

关键原则：

- `QMS` 不是单纯检验系统，而是质量裁决系统。

### 8. SRM

SRM 主责：

- 供应商协同
- 采购申请与采购订单协同
- 价格与到货协同

### 9. 设备 / IoT

设备与 IoT 主责：

- 设备台账
- 点检保养
- 故障维修
- 实时采集
- 停机与告警事件

### 10. MDM

MDM 主责：

- 主数据标准
- 编码规则
- 同步与治理

## 四、核心对象

当前平台重点围绕以下对象建设：

1. 销售订单
2. 计划需求
3. 排产任务
4. 生产工单
5. 工序任务
6. WIP 批次
7. 投料 / 消耗记录
8. 检验任务
9. 不合格处理单
10. 版本对象
11. 设备事件

当前版本控制粒度锁定为：

```text
工单 + 工序任务 + WIP 批次
```

说明：

- 当前版本不默认做到单件 SN 级追溯。
- 但结构上预留后续向 SN 级扩展的能力。

## 五、平台运行机制

平台统一采用以下运行机制：

```text
事件驱动 + 状态控制 + 审批放行
```

### 1. 事件驱动

平台的关键变化不靠人工记忆驱动，而靠事件触发：

- 销售订单变更
- 计划重算
- BOM / 工艺版本发布
- 缺料
- 到货
- 设备停机
- 检验结论
- 审批通过

### 2. 状态控制

平台通过状态机约束现场对象，而不是只靠流程图串联：

- 待齐套
- 可开工
- 生产中
- 暂停
- 异常锁定
- 待检
- 待审批
- 已放行
- 已完工
- 已关闭

### 3. 审批放行

审批的本质不是留痕，而是“允许某个状态变更合法生效”。

审批重点用于：

- 特采放行
- 报废确认
- 替代料启用
- 异常解锁
- 强制关闭
- 版本切换放行

## 六、核心控制原则

### 1. 先定义对象，再定义流程

流程依附于对象，不应脱离对象单独存在。

### 2. MES 是执行中枢，但不是一切主数据拥有者

MES 控执行，外围系统控其主责对象。

### 3. 返工采用独立返工路径

返工不直接把对象拖回原主流程，而是生成独立返工处理链。

### 4. 版本发布与版本切换分离

新版本发布后，还要识别影响范围、组织切换、控制旧版收口。

### 5. 异常先锁定，再处置，再放行

异常对象不能先流转后补救，必须先进入受控状态。

### 6. 扰动分层处理

计划扰动不直接一律全局重排，而是分级处理：

- `L1` 现场微调
- `L2` 局部重排
- `L3` 全局重排

## 七、四大业务闭环

### 1. 正向生产闭环

```text
客户需求
-> CRM 销售订单
-> ERP 计划需求 / MRP
-> APS 排程
-> PLM 提供 BOM / 工艺版本
-> WMS 备料 / 到料
-> MES 齐套校验
-> MES 工单下达
-> MES 工序执行
-> MES 报工 / WIP 流转 / 投料消耗
-> QMS 检验放行
-> WMS 入库 / 配送 / 发货
-> ERP 成本归集
-> CRM 交付闭环
```

关键控制点：

- 工单下达放行
- 工序开工放行
- 转序放行
- 完工确认放行
- 入库放行

### 2. 质量异常闭环

```text
来料 / 过程 / 成品 / 客诉发现异常
-> QMS 形成检验结论与不合格单
-> MES 锁定工单 / 工序 / WIP
-> WMS 隔离相关批次
-> QMS 组织返工 / 报废 / 特采评审
-> MES 执行返工路径或等待放行
-> QMS 复检确认
-> 恢复执行或关闭问题
```

关键原则：

- 先隔离，再评审，再处置
- 特采属于审批后放行
- 复检通过前不得自动恢复

### 3. 工程变更闭环

```text
PLM 发起 ECR / ECO
-> 工程 / 工艺 / 制造 / 质量联合评审
-> 发布新版本 BOM / 工艺 / 标准
-> 同步到相关系统
-> 识别受影响工单 / WIP / 库存 / 检验标准
-> 组织版本切换审批
-> 执行新旧版本切换
-> 完成旧版收口与归档追溯
```

关键原则：

- 版本发布不等于切换完成
- 切换影响对象必须可识别、可审批、可追溯

### 4. 计划扰动闭环

```text
插单 / 缺料 / 停机 / 批量不良 / 人员缺岗
-> MES 识别影响范围
-> APS 形成微调或重排建议
-> WMS / SRM / QMS / 设备提供约束信息
-> MES 调整工单优先级、拆分、暂停或恢复
-> 下发新的执行计划
-> 恢复生产
```

关键原则：

- 优先局部处理
- 避免所有扰动一律全局重排

## 八、跨系统关键协同场景

### 1. 缺料协同

```text
MES 识别缺料
-> WMS 校验库存
-> SRM / 采购补齐
-> 工单进入缺料锁定
-> 到料后重新齐套
-> 恢复执行
```

### 2. 设备故障协同

```text
IoT 发现停机 / 告警
-> MES 识别受影响工序与工单
-> 工序暂停或锁定
-> 设备维修任务启动
-> 维修完成确认
-> 审批或规则恢复生产
```

### 3. 批量不良协同

```text
QMS 判定批量不良
-> MES 锁定 WIP / 工序
-> WMS 隔离批次
-> QMS 形成处置结论
-> MES 执行返工或关闭
-> 复检通过后恢复
```

### 4. 插单协同

```text
CRM 订单变更或插单
-> ERP / APS 重算计划
-> MES 调整优先级与执行顺序
-> WMS 调整配送节奏
-> 更新交付承诺
```

## 九、菜单架构与业务流程的对应关系

当前前端菜单已按以下结构与业务闭环对应：

1. `首页工作台`
   用于平台全局概览与待办入口。

2. `MES 执行中枢`
   对应工单、工序、WIP、报工、追溯、异常闭环。

3. `工程与计划`
   对应 BOM、工艺路线、ECN、APS、MRP、扰动重排。

4. `协同支撑`
   对应 WMS、QMS、SRM、设备与 IoT。

5. `客户与经营`
   对应客户、销售订单、合同、应收与订单变更。

6. `管理分析`
   对应经营看板、财务、成本、能源、安环、人资。

7. `平台基础`
   对应用户、角色、菜单、审批、编码、通知、打印、SSO 等底座能力。

## 十、当前版本结论

当前版本已经明确以下平台基线：

- `MES` 执行中枢
- `工单 + 工序任务 + WIP 批次` 控制粒度
- 返工独立路径
- `领料不等于消耗`
- `QMS` 负责质量裁决
- 版本发布与版本切换分离
- 异常分级恢复
- 计划扰动分层处理
- 事件驱动 + 状态控制 + 审批放行

后续所有文档、菜单设计、页面建设、业务细化与功能调整，均应以本版本为基线，不再回到旧的模块并列式描述方式。

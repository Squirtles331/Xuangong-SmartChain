# CRUD 组件约定

## 0. 目录结构

- 每个可扩展 CRUD 组件使用“组件目录 + `index.vue`”结构。
- 当前目录建议保持：
  - `CrudPage/index.vue`
  - `CrudFormDialog/index.vue`
  - `CrudRowActions/index.vue`
  - `CrudToolbarActions/index.vue`
  - `types.ts`
  - `README.md`
- 由于当前项目解析配置原因，业务页面中优先显式引用 `index.vue`。

## 1. 工具栏按钮

- 默认使用“文字 + 图标”按钮，优先保证可发现性。
- `新增` 作为主入口动作，使用主色强调。
- `刷新`、`导出`、`返回列表` 等工具动作，使用中性样式，不与主入口抢视觉焦点。
- 批量删除、作废等高风险动作，才使用危险色。

## 2. 操作列按钮

- 默认使用紧凑的纯文字按钮，不放图标，优先保证表格密度和扫描效率。
- `编辑`、`详情`、`查看`、`配置` 等常规动作，使用主色。
- `删除` 使用危险色。
- `启用/停用`、`确认入库/确认收回`、`开始/完成` 等状态推进动作，可以按语义使用 `success` / `warning`，但同一行最多保留一个强语义强调动作。

## 3. 颜色原则

- 不采用“每种功能一个颜色”的彩虹方案。
- 采用“有限语义色”：
  - `primary`：主操作、详情、编辑、查看
  - `success`：完成、接收、通过、启用成功态
  - `warning`：停用、暂停、待确认、风险提醒
  - `danger`：删除、作废、不可逆操作
- 除删除等高风险动作外，尽量减少同一行出现多个强色按钮。

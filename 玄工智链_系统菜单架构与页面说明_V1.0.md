# 玄工智链\_系统菜单架构与页面说明\_V1.0

## 一、文档目的

本文档用于把当前前端已经落地的菜单与路由结构，整理成一份可持续维护的正式说明。

它统一回答四件事：

1. 用户在产品里实际看到的菜单叫什么
2. 菜单当前按哪些业务域拆分
3. 每个业务域承接哪些核心页面
4. 页面放在哪个菜单下，不等于它就拥有全部业务真相

本文档已按当前路由结构对齐：

- 顶层入口：`工作台`
- 业务域：`MES / PLM / APS / WMS / QMS / SRM / EAM / IoT / CRM / ERP / BI / SYS / MDM`
- 显示名称全部使用中文产品语言

## 二、使用原则

### 1. 菜单命名以用户能理解为准

- 用 `生产执行`，而不是只写 `MES`
- 用 `产品工艺`，而不是只写 `PLM`
- 用 `经营计划`，而不是把 `ERP` 直接暴露给普通用户

### 2. 菜单拆分以业务域独立维护为准

当前不再保留旧的“工程与计划 / 协同支撑 / 平台基础”混合分组，而是拆成独立业务域，方便：

- 路由单独维护
- 页面单独扩展
- 后续静态页、mock、接口按域推进

### 3. 页面归属与业务真相分开表达

页面在某个菜单下，只表示它是用户的主要入口，不表示它一定拥有全部业务真相。

例如：

- `生产领料` 放在 `仓储物流`，但它表达的是 `WMS` 库存移动
- `投料与消耗` 放在 `生产执行`，它表达的是 `MES` 现场实际消耗
- `返工裁决` 放在 `质量管理`，`返工执行` 放在 `生产执行`

### 4. 菜单真相表是命名基线

当前菜单命名与边界说明，以归档变更中的真相表为准：

- [menu-canonical-catalog.md](D:/Project/玄工智链/openspec/changes/archive/2026-07-13-realign-menu-naming-and-ownership/menu-canonical-catalog.md)

## 四、各业务域页面说明

## 4.1 工作台

- 路由：`/`
- 页面名称：`工作台`
- 页面定位：登录后的跨域行动工作台
- 页面职责：
  - 展示我的待办
  - 暴露今日异常与预警
  - 展示执行总览
  - 汇总协同域摘要
  - 提供高频快捷入口
  - 提供轻量经营摘要
- 边界说明：
  - 工作台只做聚合和导流
  - 深度经营分析统一进入 `经营分析 / 经营驾驶舱`

## 六、文档维护规则

后续新增页面或调整菜单时，必须同步检查以下四项：

1. 路由 `meta.title` 是否已使用最终中文显示名
2. 页面所属业务域是否与当前拆分方式一致
3. 隐藏页是否正确设置 `activeMenu`
4. 页面主责系统与边界说明是否仍符合菜单真相表

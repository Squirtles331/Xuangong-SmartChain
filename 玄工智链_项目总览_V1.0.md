# 玄工智链项目总览 V1.0

## 一、项目定位

玄工智链是一套面向离散制造企业的一体化制造平台，以 `MES` 为执行中枢，以事件驱动和状态控制为核心机制，以轻量外围系统完成订单、计划、工艺、物料、质量、设备、仓储与成本闭环。

项目不以模块堆叠为目标，而以“现场可控、异常可收、版本可切、责任可追、成本可归”为核心价值。

---

## 二、建设目标

### 1. 业务目标

- 建立从客户订单到生产交付的主闭环
- 建立质量异常、工程变更、计划扰动的逆向闭环
- 建立统一的工单、工序、WIP、批次、质量、设备、成本协同逻辑

### 2. 产品目标

- 明确 `MES` 执行中枢定位
- 明确外围系统协同边界
- 建立可扩展的对象模型与状态控制体系
- 支撑后续从中小工厂到中大型工厂的分阶段升级

### 3. 项目目标

- 形成统一的架构口径
- 形成可持续演进的模块地图
- 为前端页面、PRD、验收与后续开发计划提供统一基线

---

## 三、当前版本架构基线

### 1. 平台运行机制

当前版本锁定为：

```text
事件驱动 + 状态控制 + 审批放行
```

### 2. 平台最小控制粒度

当前版本锁定为：

```text
工单 + 工序任务 + WIP 批次级
```

### 3. 平台关键原则

1. `MES` 是执行中枢，但不是所有业务数据总拥有者
2. `QMS` 负责质量裁决，不只是检验
3. `WMS` 负责库存移动，`MES` 负责生产消耗
4. 返工采用独立返工路径
5. 版本发布与版本切换分离
6. 计划扰动分层处理

## 五、核心对象

平台当前围绕以下核心对象建设：

- 销售订单
- 计划需求
- 排产任务
- 生产工单
- 工序任务
- 在制品 `WIP`
- 投料/消耗记录
- 检验任务
- 不合格处置单
- 版本对象
- 设备事件

---

## 六、四大核心闭环

### 1. 正向生产闭环

客户需求 -> 计划需求 -> APS 排程 -> MES 工单执行 -> QMS 检验 -> WMS 入库/配送 -> ERP 成本归集 -> CRM 交付闭环

### 2. 质量异常闭环

异常发现 -> QMS 判定 -> MES 锁定 -> WMS 隔离 -> 返工/报废/特采 -> 复检 -> 恢复或关闭

### 3. 工程变更闭环

PLM 发版 -> 影响范围识别 -> 切换审批 -> 新旧版本收口 -> 归档追溯

### 4. 计划扰动闭环

插单/缺料/停机/批量不良 -> 影响识别 -> APS 重排 -> MES 调整执行 -> 恢复生产

## 八、当前阶段结论

玄工智链当前阶段已经完成从“模块集合思维”向“执行中枢平台思维”的转换。后续所有设计与调整都应以已确认的架构基线为准，避免回到“大而全但边界模糊”的传统一体化模式。

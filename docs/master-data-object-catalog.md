# 玄工智链主数据对象清单

## 1. 文档定位

本文档用于承接 [integrated-data-governance-blueprint.md](D:/Project/玄工智链/docs/integrated-data-governance-blueprint.md)，把“主数据治理蓝图”进一步收敛成可执行的对象清单。

本文档回答四个问题：

- 玄工智链当前必须治理哪些主数据对象
- 每类对象由哪个模块主责维护
- 哪些模块只消费，不得擅自改主字段
- 哪些对象需要版本、审批、发布机制

## 2. 使用原则

### 2.1 唯一来源

每个核心对象都必须有唯一来源模块，避免多系统各自维护一份相似数据。

### 2.2 主字段与扩展字段分离

主责模块维护主字段；消费模块只允许补充执行态、事务态、统计态字段。

### 2.3 页面先服从对象归属

后续静态页、mock、接口都应围绕对象归属设计，不以单页面自由定义字段为准。

## 3. 对象分组

玄工智链当前建议按六组管理主数据对象：

1. 基础编码与组织
2. 物料与产品定义
3. 工艺与制造资源
4. 仓储与库存基础
5. 经营协同基础
6. 质量与平台基础

## 5. 必须具备版本与发布机制的对象

以下对象不应只做普通 CRUD，必须具备明确的版本/发布概念：

- 物料
- EBOM
- MBOM
- 工艺路线
- 工序
- 检验标准
- 编码规则

建议统一具备以下状态：

- 草稿
- 待审批
- 已发布
- 已停用
- 已归档

## 6. 当前系统应新增或显式强化的对象

结合当前代码现状，以下对象应优先补齐：

### 6.1 MDM 中应新增

- 计量单位
- 物料分类
- 仓库
- 库区
- 库位
- 班次
- 产线
- 库存状态字典

### 6.2 PLM 中应强化

- MBOM 与 EBOM 区分
- 工艺路线版本管理
- ECN 生效对象范围

### 6.3 WMS 中应避免自建主档

`WMS` 不应自行维护新的物料、仓库、客户、供应商基础字段；只能消费 `MDM` 发布的数据。

### 6.4 MES 中应避免自建工程定义

`MES` 不应自行维护标准 BOM、标准工艺路线，只允许承接 `PLM` 发布后的执行基线，并记录实际执行数据。

## 7. 推荐的对象关系基线

建议静态页、mock、后续接口都遵循如下关系：

```text
物料
  -> 版本
  -> EBOM / MBOM
  -> 工艺路线
  -> 检验标准

工作中心
  -> 产线
  -> 设备
  -> 工序

仓库
  -> 库区
  -> 库位
  -> 库存状态

供应商
  -> 采购订单
  -> 收货
  -> 来料检验

客户
  -> 销售订单
  -> 生产任务
  -> 发货
```

## 8. 静态页阶段的应用方式

在“静态页先于 mock”的当前阶段，这份对象清单应用方式应为：

1. 页面标题、字段命名、关联弹窗统一按对象清单命名
2. mock 数据只能围绕对象生成，不允许每页自造近义字段
3. 新增页面前，先确认对象是否已存在于清单
4. 若对象尚未定义，先补文档，再补页面

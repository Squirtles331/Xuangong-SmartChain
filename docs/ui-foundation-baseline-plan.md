# UI Foundation Baseline Plan

## 1. 目标

本文档用于收口当前玄工智链前端静态重构阶段的 UI 基线推进方案。

当前阶段目标不是快速铺开更多静态页面，而是先把后续页面都会复用的底座稳定下来，包括：

- 系统主题方向
- Layout 结构与视觉基线
- 系统平台核心样板页规范
- 工作台首页结构与跳转规则
- 后续业务主链的静态页切入顺序

本文档定位为当前阶段的前端基线推进方案，不作为团队排期表使用。

## 2. 当前范围

本方案当前覆盖以下内容：

- 主题与配色方向
- Layout 结构与视觉规则
- 系统平台核心样板页顺序
- 工作台首页结构与插入时机
- 两周推进建议
- 后续业务主链静态页切入顺序

本方案暂不覆盖以下内容：

- 全量业务页面实现
- mock 接口接入
- API 联调
- 全部系统平台页面收尾
- 全站深色主题完整交付

## 3. 基本原则

1. 先定基线，再改代码。
2. 先做样板，再铺业务页。
3. 先静态页面和静态数据，后 mock。
4. 统一使用中文产品语言，不再回退到系统缩写命名。
5. 页面入口归属与业务真相分开表达。
6. 主题气质以工业蓝灰为主，不再延续模板化演示主题。

## 4. 主题与 Layout 基线

## 4.1 主题方向

当前正式主主题建议采用：

- `工业蓝灰`

当前可选辅主题建议采用：

- `夜班深蓝`

当前不再保留为正式产品路线：

- 多套深色主题并列
- 自由主色随意切换
- 模板化亮蓝主主题
- 演示型彩色预设主题

主题关键词建议固定为：

- 专业
- 稳定
- 工业感
- 克制
- 长时间使用不累

## 4.2 主题结构策略

当前主题策略建议理解为三层：

```text
Layout 定品牌壳
工作台定入口节奏
CRUD 页定操作效率
```

视觉强度递减关系建议为：

```text
侧边栏 > 工作台重点模块 > 普通页面 > 表格内容
```

## 4.3 Layout 结构

当前正式主布局建议为：

- `classic`

当前仅保留给特殊场景的布局为：

- `embedded`

当前不再作为正式主模式继续扩展：

- `vertical`
- `horizontal`
- `mixed`
- `columns`

## 4.4 导航结构

导航结构建议与当前路由拆分方式保持同构：

- 一级导航直接对应业务域
- 二三级导航直接对应业务域内部菜单
- 不再回退到旧的混合域分组

结构目标示意：

```text
系统平台
├─ 用户管理
├─ 角色管理
├─ 菜单管理
├─ 字典
└─ 参数
```

## 4.5 顶栏能力

当前建议保留的高频通用能力：

- 面包屑
- 用户下拉
- 通知
- 全屏
- 全局搜索
- 轻量设置入口

当前建议弱化或移除：

- 多布局切换
- 过多演示型开关
- 模板展示型主题能力

## 4.6 Tabs 策略

当前建议保留：

- 轻量标签页

建议保留的能力：

- 首页固定
- 当前页高亮
- 页签可关闭
- 最近访问快速切换

当前不建议先做：

- 拖拽排序
- 多工作区
- 复杂缓存配置
- 页签平台化高级能力

## 4.7 滚动与内边距

当前建议统一采用：

```text
layout 负责高度
主内容区负责滚动
page-layout 负责统一内边距
页面自身不重复造整页滚动壳
```

当前目标是避免以下问题继续扩散：

- 工作台不能正常滚动
- 页面内边距不一致
- 页面内容区重复包壳
- 不同页面滚动行为不统一

## 5. 系统平台核心样板页顺序

系统平台核心样板页当前建议顺序如下：

1. 用户管理
2. 参数
3. 角色管理
4. 菜单管理
5. 字典
6. 审批流

建议切分为两批推进：

```text
第一批：
用户管理 -> 参数 -> 角色管理

第二批：
菜单管理 -> 字典 -> 审批流
```

## 5.1 用户管理

定位：

- 第一张标准 CRUD 样板页

需要借这页定下的规范：

- CRUD 页骨架
- 查询区规范
- 工具栏规范
- 操作列规范
- 状态标签规范
- Dialog 表单规范
- 页头副说明规范

页头建议：

- 保留副说明

副说明建议文案：

- `维护系统账号、角色归属与启停状态`

## 5.2 参数

定位：

- 第一张配置型 CRUD 样板页

需要借这页定下的规范：

- 配置型列表页字段密度
- 参数值、说明、分类展示方式
- 配置型页面动作层级
- 标准 CRUD 与配置型 CRUD 的差异边界

页头建议：

- 副说明 + 轻归属

## 5.3 角色管理

定位：

- 授权型页面样板

需要借这页定下的规范：

- 授权型页面如何继承标准 CRUD 骨架
- 授权配置部分复杂度控制
- 角色页动作分层

页头建议：

- 保留副说明

## 5.4 菜单管理

定位：

- 树 + 列表 + 结构编辑的样板页

需要借这页定下的规范：

- 左树右表模式是否作为正式复杂页模式保留
- 结构型页面的滚动与高度策略
- 结构页工具栏与表区层次

页头建议：

- 副说明 + 轻归属

## 5.5 字典

定位：

- 主表 + 子项管理的复合页样板

需要借这页定下的规范：

- 复合页的主表与子管理入口组织方式
- 多 dialog 页面规则
- 二级管理入口的表达方式

页头建议：

- 保留副说明

## 5.6 审批流

定位：

- 复杂配置页样板

需要借这页定下的规范：

- 流程型配置页表达方式
- 步骤条与节点表达层级
- 复杂配置页如何保持统一骨架

页头建议：

- 副说明 + 轻归属

## 6. 工作台插入时机

工作台不建议作为第一张样板页先做，也不建议拖到系统平台全部完成后再做。

当前更合理的插入时机建议为：

```text
主题基线
-> Layout 基线
-> 用户管理
-> 参数
-> 角色管理
-> 工作台
-> 菜单管理 / 字典 / 审批流
```

原因：

- 前 3 张样板页先把标准列表页语言定住
- 工作台随后落地，不会过早干扰全站页面规范
- 复杂后台页再后置，更容易保持统一

## 6.1 工作台结构基线

工作台当前建议固定为六大模块：

1. 我的待办
2. 今日异常与预警
3. 执行总览
4. 协同域摘要
5. 快捷入口
6. 经营摘要

首页首屏建议优先承接：

- 我的待办
- 今日异常与预警
- 执行总览

工作台定位固定为：

> 登录后的跨域行动工作台

工作台与经营驾驶舱边界固定为：

- 工作台回答：`现在要做什么`
- 经营驾驶舱回答：`最近经营表现怎么样`

## 7. 两周推进建议

## 7.1 第 1 周

第 1 周建议以讨论、收口和样板页规则确认为主，不进行大规模实现。

建议收口内容：

1. 主题方向
2. Layout 结构
3. Layout 视觉与滚动规则
4. 用户管理样板页
5. 参数样板页
6. 角色管理样板页
7. 工作台结构与跳转规则
8. 菜单管理结构页规则
9. 字典复合页规则

第 1 周最重要的产出不是代码数量，而是：

- 可执行的结论
- 清晰的样板页规则
- 可直接进入实现的第 2 周顺序

## 7.2 第 2 周

第 2 周再按已拍板结果集中改代码。

建议实现顺序：

1. 主题基线
2. Layout 基线
3. 用户管理
4. 参数
5. 角色管理
6. 工作台
7. 菜单管理
8. 字典（如时间允许）

当前不建议第 1 周就直接大规模改代码，避免出现以下返工：

- 主题未定先改 Layout
- Layout 未定先改样板页
- 样板页未定先铺业务页
- 工作台先做过深后反向干扰标准页规范

## 8. 完成标准

满足以下条件，可视为当前 UI 基线阶段完成：

- 正式主主题方向确定
- Layout 主模式确定
- 导航结构确定
- Tabs 策略确定
- 页面滚动与内边距规则确定
- 标准 CRUD 页样板确定
- 配置型页样板确定
- 授权型页样板确定
- 工作台结构确定
- 结构型后台页样板确定
- 复合页样板确定

## 9. 下一阶段

当前 UI 基线阶段完成后，下一阶段建议切入业务主链静态页，优先顺序如下：

1. 工单列表
2. 我的任务
3. 工序任务
4. 在制品
5. 检验任务
6. 异常处置
7. 生产领料
8. 不合格处置

下一阶段仍然建议坚持：

```text
先静态页
先静态数据
后 mock
后接口
```

## 10. 结论

当前玄工智链前端推进的关键，不是继续平均铺开更多页面，而是先把主题、Layout、样板页和工作台这些后续高复用基线稳定下来。

当前建议的推进主线为：

```text
主题基线
-> Layout 基线
-> 系统平台轻样板
-> 工作台
-> 系统平台复杂样板
-> MES / QMS / WMS 主链
```

只要先把这条主线走稳，后续业务页面的静态重构、mock 接入和接口联调都会顺很多，返工面也会显著缩小。

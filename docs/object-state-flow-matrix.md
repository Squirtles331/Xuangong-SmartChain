# 玄工智链对象状态流矩阵

## 1. 文档定位

本文档用于定义玄工智链关键对象的状态流和跨模块前置约束，作为后续静态页交互、mock 校验、接口前置校验和按钮可操作性的统一依据。

本文档重点解决三个问题：

- 对象有哪些标准状态
- 状态如何流转
- 哪些状态变化必须依赖上游对象或下游反馈

## 2. 使用原则

### 2.1 状态不是展示文案，而是业务约束

状态必须决定：

- 页面按钮是否可用
- 是否允许进入下一步
- 是否允许被下游系统消费

### 2.2 状态流必须和对象主责一致

- `PLM` 负责定义和发布工程基线
- `MES` 负责执行真相
- `WMS` 负责库存事务真相
- `QMS` 负责质量裁决真相
- `ERP` 负责经营与财务账真相

### 2.3 当前阶段先固化规则，再逐步实现联动

即使当前阶段暂未完成真实 API 联动，也必须先把状态流体现到：

- 静态页按钮逻辑
- mock 数据状态变化
- 后续接口设计

## 3. 通用状态层次

不同对象允许有不同状态，但整体应尽量落入以下层次：

| 状态层次        | 含义                     | 常用值                        |
| --------------- | ------------------------ | ----------------------------- |
| 草稿态          | 尚未形成正式业务对象     | `draft`                       |
| 审批态          | 正在走审核流程           | `pending_approval`            |
| 已批准态        | 已被批准，待发布或待执行 | `approved`                    |
| 已发布/已下达态 | 已可被下游消费或执行     | `released` `active` `enabled` |
| 执行中          | 现场或流程正在推进       | `in_progress`                 |
| 已完成          | 主体业务已完成           | `completed`                   |
| 已关闭          | 完成结案或人工终止       | `closed`                      |
| 已停用          | 不再继续使用，但可查阅   | `disabled`                    |
| 已归档          | 历史冻结，不可再改       | `archived`                    |

## 4. 关键对象状态流

### 4.1 物料

| 状态               | 含义     | 允许操作                     | 禁止操作             | 前置条件                     |
| ------------------ | -------- | ---------------------------- | -------------------- | ---------------------------- |
| `draft`            | 草稿物料 | 编辑、提交审批、删除         | 被 BOM/工单/库存引用 | 无                           |
| `pending_approval` | 待审批   | 审批、驳回                   | 下游正式使用         | 基础字段已完整               |
| `active`           | 已生效   | 被 BOM、工单、采购、库存引用 | 任意改主字段         | 审批通过并发布               |
| `disabled`         | 已停用   | 查询、恢复                   | 新建业务引用         | 无在途关键事务或人工确认豁免 |
| `archived`         | 已归档   | 查询                         | 恢复普通使用         | 已彻底退出业务使用           |

推荐流转：

`draft -> pending_approval -> active -> disabled -> archived`

### 4.2 BOM

| 状态               | 含义     | 允许操作               | 禁止操作           | 前置条件               |
| ------------------ | -------- | ---------------------- | ------------------ | ---------------------- |
| `draft`            | 草稿 BOM | 编辑、提交审批、删除   | 生产领料、正式排程 | 母件物料已存在         |
| `pending_approval` | 待审批   | 审批、驳回             | 执行引用           | 结构完整、关键子项齐全 |
| `active`           | 生效 BOM | 被工单、排程、领料引用 | 直接改结构         | 审批通过并发布         |
| `disabled`         | 停用 BOM | 查询、复制新版本       | 新工单引用         | 无新工单继续使用该版本 |
| `archived`         | 历史归档 | 查询对比               | 恢复执行使用       | 已被新版本替代         |

推荐流转：

`draft -> pending_approval -> active -> disabled -> archived`

关键约束：

- `EBOM` 可转化为 `MBOM`，但不能自动等价替代
- `MBOM active` 后，MES 才能用其生成正式领料口径
- BOM 版本切换必须通过 ECN 或正式发布动作完成

### 4.3 工艺路线

| 状态               | 含义     | 允许操作             | 禁止操作         | 前置条件                     |
| ------------------ | -------- | -------------------- | ---------------- | ---------------------------- |
| `draft`            | 草稿工艺 | 编辑、提交审批、删除 | 工单正式引用     | 物料与工序结构已基本形成     |
| `pending_approval` | 待审批   | 审批、驳回           | 下达执行         | 工时、工作中心、工序关系完整 |
| `enabled`          | 已启用   | 被工单、排程引用     | 直接修改工序结构 | 审批通过并发布               |
| `disabled`         | 已停用   | 查询、复制版本       | 新工单引用       | 无新工单继续按该版本下达     |
| `archived`         | 已归档   | 查询                 | 恢复执行使用     | 已有替代版本                 |

推荐流转：

`draft -> pending_approval -> enabled -> disabled -> archived`

### 4.4 ECN

| 状态               | 含义         | 允许操作                     | 禁止操作           | 前置条件           |
| ------------------ | ------------ | ---------------------------- | ------------------ | ------------------ |
| `draft`            | 草稿变更     | 编辑、补充影响对象、提交审批 | 直接影响执行侧     | 变更原因与范围明确 |
| `pending_approval` | 待审批       | 审批、驳回                   | 发布               | 影响对象清单完整   |
| `approved`         | 已批准待生效 | 排定生效、推送相关域         | 直接关闭           | 审批完成           |
| `effective`        | 已生效       | 查询、追踪影响               | 再次编辑主变更内容 | 已完成下游同步     |
| `closed`           | 已关闭       | 查询                         | 再次生效           | 生效验证完成       |

推荐流转：

`draft -> pending_approval -> approved -> effective -> closed`

关键约束：

- ECN 未生效前，BOM/工艺新版本不能默认进入 MES 执行面
- 若已有在制工单，必须明确是“新工单生效”还是“在制切换”

### 4.5 工单

| 状态          | 含义     | 允许操作                 | 禁止操作         | 前置条件                 |
| ------------- | -------- | ------------------------ | ---------------- | ------------------------ |
| `draft`       | 草稿工单 | 编辑、提交审核、删除     | 开工、领料、报工 | 上游订单/BOM/工艺已齐备  |
| `approved`    | 已审核   | 下达、撤回               | 直接报工         | 审核通过                 |
| `released`    | 已下达   | 开工、领料、派工         | 关闭结案         | MES 接收到正式任务       |
| `in_progress` | 生产中   | 报工、异常、领退料、质检 | 删除             | 已发生现场执行           |
| `completed`   | 已完工   | 完工入库、结案申请       | 再次正常报工     | 全部工序完成             |
| `closed`      | 已关闭   | 查询追溯                 | 继续执行         | 入库、质量、结算条件满足 |

推荐流转：

`draft -> approved -> released -> in_progress -> completed -> closed`

关键约束：

- 未 `approved` 不得 `released`
- 未 `released` 不得开工报工
- 未 `completed` 不得作为完工入库来源
- 未完工但需要结束时，应有单独异常关闭/强制关闭机制

### 4.6 报工记录

| 状态        | 含义         | 允许操作             | 禁止操作     | 前置条件     |
| ----------- | ------------ | -------------------- | ------------ | ------------ |
| `draft`     | 草稿报工     | 编辑、提交           | 计入正式产量 | 工单已下达   |
| `submitted` | 已提交待确认 | 审核、退回           | 直接结算     | 报工数据完整 |
| `confirmed` | 已确认       | 推送 WIP、产量、工时 | 修改关键字段 | 审核通过     |
| `rejected`  | 已退回       | 重新编辑提交         | 进入统计口径 | 审核退回     |

推荐流转：

`draft -> submitted -> confirmed`

异常流转：

`submitted -> rejected -> draft`

### 4.7 收货/入库单

| 状态        | 含义          | 允许操作         | 禁止操作     | 前置条件       |
| ----------- | ------------- | ---------------- | ------------ | -------------- |
| `draft`     | 草稿单据      | 编辑、提交、删除 | 计入库存     | 来源单存在     |
| `pending`   | 待入库/待处理 | 入库、取消       | 直接完成账务 | 单据已确认     |
| `completed` | 已入库        | 查询、冲销申请   | 再次入库     | 库存事务已写入 |
| `cancelled` | 已取消        | 查询             | 再次处理     | 未完成库存写入 |

推荐流转：

`draft -> pending -> completed`

异常流转：

`draft -> cancelled`
`pending -> cancelled`

关键约束：

- 来料收货若要求检验，`completed` 前应有质量裁决条件
- 生产入库应来源于 `工单 completed`

### 4.8 检验记录

| 状态        | 含义         | 允许操作                  | 禁止操作       | 前置条件     |
| ----------- | ------------ | ------------------------- | -------------- | ------------ |
| `draft`     | 草稿检验     | 录入结果、提交            | 裁决库存状态   | 检验任务存在 |
| `submitted` | 已提交待裁决 | 判定、退回                | 自动转可用库存 | 检验结果完整 |
| `accepted`  | 判定合格     | 放行、入可用库存          | 改关键结果     | 裁决通过     |
| `rejected`  | 判定不合格   | 触发不合格处置、返工/退货 | 直接入可用库存 | 裁决不通过   |
| `closed`    | 已关闭       | 查询                      | 再次编辑       | 后续处置完成 |

推荐流转：

`draft -> submitted -> accepted -> closed`

异常流转：

`draft -> submitted -> rejected -> closed`

### 4.9 库存台账记录

库存台账通常不是“人工状态流”对象，而是事务结果对象。建议仅定义记录状态：

| 状态       | 含义   | 说明            |
| ---------- | ------ | --------------- |
| `posted`   | 已记账 | 正常库存事务    |
| `reversed` | 已冲销 | 对原事务冲销    |
| `frozen`   | 已冻结 | 受质量/业务控制 |

## 5. 跨模块强约束矩阵

### 5.1 关键业务约束

| 上游对象状态                    | 下游动作       | 是否允许 | 说明                 |
| ------------------------------- | -------------- | -------- | -------------------- |
| 物料 `draft`                    | 建 BOM         | 否       | 只能引用已生效物料   |
| BOM `draft`                     | 下达工单       | 否       | 必须引用已生效 BOM   |
| 工艺路线 `draft`                | 下达工单       | 否       | 必须引用已启用工艺   |
| 工单 `draft`                    | 报工           | 否       | 工单必须已下达       |
| 工单 `approved`                 | 报工           | 否       | 审核通过不等于已下达 |
| 工单 `released`                 | 报工           | 是       | 正常执行入口         |
| 工单 `in_progress`              | 领料           | 是       | 执行中允许继续领退料 |
| 工单 `completed`                | 成品入库       | 是       | 作为生产入库来源     |
| 检验记录 `accepted`             | 批次转可用库存 | 是       | 合格放行             |
| 检验记录 `rejected`             | 批次转可用库存 | 否       | 必须进入不合格处置   |
| ECN `approved` 但未 `effective` | MES 切换新版本 | 否       | 未生效不得执行       |
| 收货单 `completed`              | ERP 记库存账   | 是       | 入库完成后方可记账   |

## 6. 当前阶段页面需要直接体现的按钮规则

在静态页阶段，建议直接把以下规则体现在按钮可用性上：

### 6.1 BOM 页面

- `draft` 可编辑、提交审批、删除
- `active` 可查看、复制新版本、发起变更
- `archived` 仅可查看、对比

### 6.2 工艺路线页面

- `draft` 可编辑工序
- `enabled` 不允许直接改结构，只能复制版本
- `disabled` 不允许作为新工单基线

### 6.3 工单页面

- `draft` 显示“提交审核”
- `approved` 显示“下达”
- `released`/`in_progress` 显示“报工”“异常”“领退料”
- `completed` 显示“完工入库”
- `closed` 仅保留“详情/追溯”

### 6.4 收货/入库页面

- `draft` 可编辑删除
- `pending` 可执行入库
- `completed` 不可再编辑，只可冲销或查看

## 7. 当前阶段最该先做的状态流对象

从投入产出比看，建议优先把以下五类对象的状态流先做完整：

1. 物料
2. BOM
3. 工艺路线
4. 工单
5. 收货/入库单

质量记录和 ECN 作为第二批补齐。

## 8. 对 mock 设计的直接要求

后续 mock 不能只返回静态表格数据，必须体现状态差异：

- 至少准备每个对象 3-5 种不同状态样本
- 不同状态下按钮可操作性不同
- 不允许所有行都处于“可编辑”或“已完成”单一状态

## 9. 下一步建议

完成本文档后，下一步最合适的是：

- 把高优先对象的静态页按状态流补全按钮规则
- 再据此生成 mock 状态样本
- 最后再进入 API 状态机对齐

# 玄工智链页面与对象映射表

## 1. 文档定位

本文档用于把当前系统中的业务页面，映射到统一对象体系上，作为后续静态页开发、mock 设计、接口拆分和联调排期的直接依据。

它与前两份文档的关系是：

- [integrated-data-governance-blueprint.md](D:/Project/玄工智链/docs/integrated-data-governance-blueprint.md)
  负责回答“为什么现在要先统一对象和主数据”
- [master-data-object-catalog.md](D:/Project/玄工智链/docs/master-data-object-catalog.md)
  负责回答“系统里有哪些核心对象、谁主责”
- 本文档
  负责回答“现有页面分别落在哪些对象上，应该按什么顺序推进”

## 2. 使用规则

### 2.1 页面必须先映射对象，再定义字段

任何新页面、新 mock、新接口，在开始实现前都应先回答：

- 它服务于哪个核心对象
- 这个对象属于哪一层
- 哪个模块拥有该对象真相
- 当前页面是在维护对象、消费对象，还是处理对象事务

### 2.2 一个页面可以关联多个对象，但只能有一个主对象

例如：

- `工单详情` 会关联工单、物料、工艺路线、报工、领料、异常
- 但主对象仍然是 `工单`

### 2.3 页面优先级服从业务主链优先级

不是页面越多越优先，而是越接近一体化主链越优先。

## 3. 对象层级标签

为便于后续统一表达，页面映射统一使用以下对象层级：

- `主数据层`
- `产品与工程定义层`
- `计划与交易层`
- `执行与追溯层`
- `分析与聚合层`
- `平台与治理层`

## 6. 页面类型说明

为后续统一交互和 mock 设计，建议把页面分为以下几类：

- `主档维护页`
- `主对象列表页`
- `主对象编辑页`
- `复合详情页`
- `执行事务页`
- `规则配置页`
- `查询台账页`
- `分析页`
- `聚合看板页`

这能帮助后续统一：

- 页面骨架
- 工具栏动作
- 表格列风格
- 数据密度
- mock 数据结构

## 8. 页面映射在当前阶段的落地方式

这份矩阵在当前阶段的正确用法是：

1. 开发某页面前先查主对象
2. 再查该对象的字段口径文档
3. 再决定该页是主档页、事务页还是分析页
4. mock 数据优先按对象组织，而不是按页面拼凑
5. 接口拆分优先按对象和事务边界拆，不按前端目录随意拆

# 玄工智链 · XIC Platform

> 面向离散制造企业的，以 `MES` 为执行中枢、以事件驱动和状态控制为核心机制、以轻量外围系统完成协同闭环的一体化制造平台。

---

## 平台定位

玄工智链（XIC Platform）不是一个单纯堆叠模块的“大而全平台”，而是一套围绕制造现场执行深度建设的离散制造协同平台。

平台坚持以下定位：

- `MES` 作为统一执行中枢，主导工单、工序、在制品、报工、投料、追溯与异常联动。
- `ERP / CRM / SRM / PLM / WMS / QMS / SCADA / MDM` 按业务域分工协同，不重复建设、不重叠拥有。
- 平台核心价值不在“模块很多”，而在“现场可控、异常可收、版本可切、责任可追、成本可归”。

一句话概括：

> `MES` 强中枢，不做全拥有者；平台靠事件驱动执行，靠状态控制现场。

---

## 架构总述

平台整体采用“重执行、轻外围、强闭环、可扩展”的架构思路：

- 上游由 `CRM`、`ERP`、`SRM` 承接客户需求、经营计划、采购协同与成本口径。
- 中游由 `PLM`、`APS`、`MDM` 承接产品定义、工艺路线、版本合法性、排程建议与主数据治理。
- 核心由 `MES` 作为执行中枢，统一维护工单、工序、WIP、报工、投料、生产日志与异常状态。
- 两翼由 `WMS`、`QMS`、`SCADA/IoT` 分别提供库存实物真相、质量裁决真相与设备原始事件输入。
- 下游由 `BI / Analysis` 汇总形成成本、质量、设备、交付、OEE 与经营分析。

```text
经营与客户层
CRM / ERP / SRM

工程与计划层
PLM / APS / MDM

执行中枢层
MES

协同支撑层
WMS / QMS / SCADA-IoT / 设备维护

分析与决策层
BI / Dashboard / Cost / OEE / Trace
```

## 最终确认项

当前版本按以下架构基线建设，后续模块调整、README 描述与产品口径统一以此为准：

1. `MES` 是执行中枢，但不是所有业务数据的总拥有者。
2. 控制粒度锁定为：`工单 + 工序任务 + WIP 批次级`。
3. 返工采用独立返工路径，不直接揉回原工单主流程。
4. `领料不等于消耗`：
   `WMS` 负责库存移动真相，`MES` 负责生产消耗真相。
5. `QMS` 负责检验结论与不合格处置裁决，不只是检验记录。
6. `版本发布 != 版本切换完成`，版本切换是受控过程。
7. 异常恢复分级：
   一般异常可规则恢复，关键异常必须审批恢复。
8. 计划扰动按 `L1 现场微调 / L2 局部重排 / L3 全局重排` 分层处理。
9. 平台运行机制锁定为：`事件驱动 + 状态控制 + 审批放行`。

## 系统边界

平台采用“一个对象一个主真相源”的边界原则：

| 系统 | 主责范围 |
| --- | --- |
| `CRM` | 客户、订单承诺、交付跟踪、售后与客诉入口 |
| `ERP` | 计划总量、MPS/MRP、成本归集、财务口径 |
| `APS` | 有限能力排程、局部/全局重排建议 |
| `PLM` | 产品定义、BOM、工艺路线、版本合法性 |
| `MES` | 工单、工序、WIP、报工、投料、执行状态、异常联动 |
| `WMS` | 库存、库位、批次、出入库、线边配送 |
| `QMS` | 检验任务、不合格处置、特采/报废/返工裁决 |
| `SCADA / IoT` | 设备原始事件、参数采集、停机/报警输入 |
| `MDM` | 主数据标准、编码、同步与治理 |

边界原则：

- `MES` 拥有执行真相，不越权改写订单、版本、库存、质量裁决主数据。
- `WMS` 不直接主导工单执行状态，但其库存事件可影响开工放行与缺料锁定。
- `QMS` 不直接主导生产执行，但其质量结论可锁定或放行工单、工序与 WIP。
- `PLM` 负责版本合法性，`MES/WMS/QMS` 负责版本切换后的执行落地。

## 核心控制原则

### 1. 先定义对象，再定义流程

平台重点管理的核心对象包括：

- 销售订单
- 计划需求
- 排产任务
- 生产工单
- 工序任务
- 在制品 `WIP`
- 投料/消耗记录
- 检验任务
- 不合格处置单
- 版本对象
- 设备事件

### 2. 平台靠状态机运转，不靠流程箭头运转

平台统一管理以下关键状态：

- 待齐套
- 可开工
- 生产中
- 暂停
- 异常锁定
- 待检
- 待审批
- 已放行
- 已完工
- 已关闭

### 3. 异常先锁定，再处置，再放行

平台统一执行以下控制原则：

- 缺料、停机、质量异常、版本切换、关键资质缺失等场景先锁定对象。
- 锁定层级可到工单、工序、WIP、批次。
- 解锁依赖规则恢复或审批恢复，不能仅靠人工经验继续流转。

### 4. 审批的本质是合法改写状态

审批不是简单留痕，而是合法触发以下动作：

- 工单/工序/WIP 解锁
- 特采放行
- 替代料生效
- 返工、报废、强制关闭
- 版本切换放行

## 四大业务闭环

### 1. 正向生产闭环

```text
客户需求 -> CRM/ERP -> APS排程 -> MES工单下达
-> 齐套校验 -> 工序执行 -> WIP流转 -> 检验放行
-> WMS入库/配送 -> ERP成本归集 -> CRM交付闭环
```

### 2. 质量异常闭环

```text
来料/过程/成品/客诉异常
-> QMS检验与判定
-> MES锁单/锁工序/WIP冻结
-> WMS隔离批次
-> 返工/报废/特采评审
-> 复检
-> 恢复或关闭
```

### 3. 工程变更闭环

```text
PLM发起ECR/ECO
-> 版本评审
-> 发布新定义
-> 识别受影响工单/WIP/库存
-> 切换审批与执行
-> 新旧版本收口
-> 归档追溯
```

### 4. 计划扰动闭环

```text
插单/缺料/停机/批量不良/人员缺岗
-> MES识别影响范围
-> APS微调/局部重排/全局重排
-> WMS/SRM/QMS协同约束
-> 新计划下发
-> 恢复执行
```

## 事件驱动机制

平台不是静态流程系统，而是事件驱动的执行控制系统。

主要事件来源包括：

- `CRM / ERP / APS`：订单变更、交期变化、计划重算、插单请求
- `PLM`：BOM发布、工艺更新、版本切换、生效策略变化
- `WMS / SRM`：库存变化、缺料、到货、发料、配送延迟
- `QMS`：检验结论、不合格判定、特采/报废/复检结果
- `SCADA / IoT`：停机、报警、参数超限、设备恢复、换模完成
- `审批流`：特采批准、替代料批准、解锁批准、版本切换批准

平台响应逻辑：

```text
事件进入 MES
-> 识别事件类型
-> 判断影响范围
-> 改写工单/工序/WIP/锁定状态
-> 通知协同系统继续动作
-> 留痕、追溯、分析归集
```

## 模块地图

### 执行中枢

- `MES`：工单管理、工序管理、WIP、报工、齐套、投料、追溯、生产看板、生产日志

### 工程与计划

- `PLM`：产品、BOM、EBOM/MBOM、工艺路线、作业文件、ECR/ECO
- `ERP`：销售订单承接、计划需求、MPS、MRP、成本核算、基础财务
- `APS`：有限能力排程、约束配置、重排建议
- `MDM`：客户、物料、产品、设备、人员、组织、编码与同步治理

### 协同支撑

- `WMS`：入库、出库、库位、批次、线边配送、盘点、调拨、条码
- `QMS`：IQC、IPQC、FQC、检验标准、不合格处置、质量追溯
- `SRM`：供应商、询价、采购订单、到货管理、供应商评价
- `IoT / 设备维护`：设备台账、点位、实时数据、报警、维保、OEE

### 管理与分析

- `CRM`：客户档案、合同、交付跟踪、售后服务、客诉管理
- `BI / Dashboard`：生产、质量、成本、设备、OEE、综合驾驶舱
- `System`：用户、角色、菜单、编码规则、审批流、日志、通知、SSO

## 开发说明

### 文件命名

| 类型 | 规范 | 示例 |
| --- | --- | --- |
| 目录 | `kebab-case` | `work-order/`, `code-rule/` |
| 页面入口 | `index.vue` | `work-order/list/index.vue` |

### 组件使用优先级

```text
gi-component -> vxe-table -> Element Plus
```

### 页面布局约定

```vue
<gi-page-layout>
  <template #header><gi-form search /></template>
  <template #left><el-tree /></template>
  <template #tool><gi-button /></template>
  <gi-table />
  <gi-dialog />
```

- 列宽优先使用 `minWidth`
- 操作栏固定右侧：`fixed: 'right'`

# 玄工智链 XIC Platform

> 面向离散制造企业的 MES 中枢型一体化平台。

## 1. 平台定位

玄工智链不是“所有模块都自己做一遍”的大而全系统，而是一套以 `MES` 为执行中枢、以事件驱动和状态控制为核心机制、以外围专业系统协同闭环的一体化制造平台。

平台当前采用的最终确认口径如下：

1. `MES` 是执行中枢，但不是所有主数据和交易数据的唯一拥有者。
2. 平台当前最小控制粒度锁定为：`工单 + 工序任务 + WIP 批次`。
3. `返工` 采用独立返工路径，不直接把对象硬拉回原主流程。
4. `领料不等于消耗`：`WMS` 负责库存移动真相，`MES` 负责生产消耗真相。
5. `QMS` 负责质量裁决，不只是检验记录。
6. `版本发布 != 版本切换完成`，切换必须是受控过程。
7. 异常恢复分级处理：一般异常可规则恢复，关键异常必须审批恢复。
8. 计划扰动分层处理：`L1 现场微调 / L2 局部重排 / L3 全局重排`。
9. 平台运行机制锁定为：`事件驱动 + 状态控制 + 审批放行`。

## 2. 平台分层

```text
客户与经营层
CRM / 订单承诺 / 合同 / 应收

工程与计划层
PLM / BOM / 工艺路线 / APS / MRP / MDM

执行中枢层
MES / 工单 / 工序任务 / WIP / 报工 / 投料 / 追溯 / 异常

协同支撑层
WMS / QMS / SRM / 设备 / IoT

管理分析层
经营看板 / 成本 / 财务 / OEE / 能源 / 安环 / 人资

平台基础层
用户 / 角色 / 菜单 / 审批 / 编码 / 日志 / 通知 / SSO
```

## 3. 系统边界

| 系统         | 主职责                                          |
| ------------ | ----------------------------------------------- |
| `CRM`        | 客户、销售订单、合同、交付跟踪、售后入口        |
| `ERP`        | 经营计划、MPS/MRP、成本归集、财务口径           |
| `PLM`        | 产品定义、BOM、工艺路线、版本合法性、ECR/ECO    |
| `APS`        | 有限能力排程、约束配置、局部或全局重排建议      |
| `MES`        | 工单、工序、WIP、报工、投料、执行状态、异常联动 |
| `WMS`        | 库存、库位、批次、出入库、配送、盘点、条码      |
| `QMS`        | 检验任务、不合格处理、特采、返工/报废裁决       |
| `SRM`        | 采购协同、供应商协同、价格与到货协同            |
| `设备 / IoT` | 设备台账、点检保养、采集、告警、停机事件        |
| `MDM`        | 主数据标准、编码规则、同步与治理                |

## 4. 当前前端菜单架构

前端菜单与路由已按最终确认版切换为以下结构：

1. 首页工作台
2. MES 执行中枢
3. 工程与计划
4. 协同支撑
5. 客户与经营
6. 管理分析
7. 平台基础

其中：

- 已有页面保留原有实现基础，但已重新归位到新架构。
- 未开工页面统一进入“等待建设”占位页，作为后续功能建设入口。
- 隐藏详情页通过 `activeMenu` 与所属菜单保持联动高亮。

## 5. 当前目录组织

```text
src/
├─ api/                 接口层
├─ components/          通用组件
├─ layout/              布局系统
├─ mock/                Mock 数据
├─ router/              菜单与路由
├─ stores/              状态管理
├─ styles/              全局样式
├─ utils/               工具封装
└─ views/               业务页面
```

业务页面当前主要按以下目录组织：

- `work-order/`
- `bom/`
- `routing/`
- `ecn/`
- `aps/`
- `mrp/`
- `wms/`
- `qms/`
- `scm/`
- `equipment/`
- `iot/`
- `crm/`
- `analysis/`
- `finance/`
- `energy/`
- `ehs/`
- `hr/`
- `system/`
- `mdm/`
- `settings/`
- `shared/`

说明：

- 目录名暂保留现有工程结构，便于平滑演进。
- 菜单归属、业务语义、用户视角已经切换到新架构口径。
- 后续若继续演进，可再逐步做目录级重构，而不是和业务确认同步硬改。

# 以 MES 为核心的一体化工业平台 Demo 设计文档

---

# 1. 平台定位

## 1.1 产品定位

本平台定位：

> 面向制造企业的一体化工业数字化平台，以 MES（制造执行系统）为核心，ERP、PLM、WMS、QMS、CRM、SRM、IoT 等系统作为辅助能力。

整体设计原则：

* MES 功能深度建设
* 周边系统轻量化建设
* 保持业务边界清晰
* 支持未来独立扩展专业版系统

---

# 2. 整体业务架构

```
                         工业一体化平台

        CRM        SRM        IoT        BI
```

这一部分非常重要，因为你的平台定位是：

> **MES 为核心的一体化工业平台**

所以必须明确每个系统：

* 负责什么
* 核心业务对象是什么

# 4.1 MES 制造执行系统（核心系统）

## 系统定位

MES 是平台核心。

负责：

> 将企业生产计划转化为车间可执行任务，并管理生产全过程。

MES连接：

```text
经营层
ERP销售订单

        ↓

计划层
APS生产计划

        ↓

执行层
MES生产执行

        ↓

现场层
设备 / 人员 / 物料
```

---

## 核心业务对象

| 对象   | 说明       |
| ---- | -------- |
| 生产工单 | 车间执行任务   |
| 工序   | 制造过程步骤   |
| 派工单  | 人员设备分配任务 |
| 生产报工 | 实际生产记录   |
| WIP  | 在制品状态    |
| 产品追溯 | 生产全过程记录  |
| 生产异常 | 现场异常处理   |

---

## 数据输入

来源：

### ERP

提供：

* 销售订单
* 产品需求
* 交付日期

### APS

提供：

* 排产计划
* 生产优先级

### PLM

提供：

* BOM
* 工艺路线

### IoT

提供：

* 设备状态
* 设备数据

---

## 数据输出

提供给：

### WMS

* 领料需求
* 退料需求

### QMS

* 检验任务
* 生产批次

### BI

* 生产数据
* OEE数据

---

## 不负责

MES 不负责：

❌ 客户管理

❌ 销售订单

❌ 财务核算

❌ 库存账务

❌ 工程设计

# 9. AI开发约束规则

开发过程中必须遵守：

```
1.

CRM不能创建生产工单。


2.

ERP销售订单不能直接作为生产任务。


3.

MES只管理生产执行。


4.

销售订单和生产工单不是一对一关系。


5.

多个销售订单可以生成一个生产工单。


6.

BOM属于PLM。


7.

库存属于WMS。


8.

设备实时数据属于IoT。


9.

基础编码属于MDM。


10.

所有页面先实现静态展示，不开发真实业务逻辑。

```

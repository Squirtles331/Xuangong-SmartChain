# 玄工智链_核心状态机定义_V1.0

## 一、文档目的

本文档用于把当前平台“状态控制”原则落成可执行状态机定义。

目标是回答以下问题：

- 核心对象有哪些状态
- 状态之间怎么迁移
- 哪些迁移自动发生，哪些必须审批
- 哪些状态会锁住流程
- 哪些状态恢复后可以继续执行

---

## 二、状态机设计原则

### 1. 状态属于对象，不属于页面

页面只是展示和触发状态变化，不能创造独立于对象之外的伪状态。

### 2. 状态迁移必须有触发条件

不允许出现“人工手动改成任何状态”的自由跳转设计。

### 3. 锁定状态优先级高于执行状态

异常、质量、版本、缺料等锁定应能中断正常执行流。

### 4. 审批的作用是让状态迁移合法生效

审批不是备注，而是状态变更许可。

---

## 三、核心状态对象范围

当前版本重点定义以下状态机：

1. 生产工单
2. 工序任务
3. WIP 批次
4. 检验任务
5. 不合格处理单
6. 版本切换任务

---

## 四、生产工单状态机

## 4.1 状态定义

| 状态 | 含义 |
| --- | --- |
| `draft` | 草稿，尚未正式提交 |
| `pending_release` | 待下达，已完成创建但未放行 |
| `waiting_kitting` | 待齐套，等待物料和前置条件满足 |
| `ready` | 可开工，具备进入执行的条件 |
| `in_progress` | 生产中 |
| `paused` | 暂停，因一般扰动暂时中断 |
| `locked` | 异常锁定，不允许继续执行 |
| `waiting_inspection` | 待检，执行完成后等待质量放行 |
| `completed` | 已完工 |
| `closed` | 已关闭 |
| `cancelled` | 已取消 |

## 4.2 允许迁移

| 当前状态 | 目标状态 | 触发条件 | 是否审批 |
| --- | --- | --- | --- |
| `draft` | `pending_release` | 工单创建提交成功 | 否 |
| `pending_release` | `waiting_kitting` | 工单下达后进入齐套准备 | 否 |
| `waiting_kitting` | `ready` | 齐套校验通过 | 否 |
| `ready` | `in_progress` | 首道工序开工 | 否 |
| `in_progress` | `paused` | 一般扰动暂停 | 否 |
| `paused` | `in_progress` | 扰动解除且允许恢复 | 否 / 视规则 |
| `in_progress` | `locked` | 质量、缺料、版本、设备等关键异常 | 否，自动锁定 |
| `locked` | `in_progress` | 解锁审批通过 | 是 |
| `in_progress` | `waiting_inspection` | 最终执行完成待检 | 否 |
| `waiting_inspection` | `completed` | 检验放行通过 | 否 |
| `completed` | `closed` | 入库、归档、收口完成 | 否 |
| `pending_release` / `waiting_kitting` | `cancelled` | 未开工取消 | 是 |
| `in_progress` / `paused` / `locked` | `closed` | 强制关闭 | 是 |

## 4.3 禁止迁移

- `draft -> in_progress`
- `waiting_kitting -> completed`
- `locked -> completed`
- `waiting_inspection -> in_progress`（除非返工分支另建对象）

## 五、工序任务状态机

## 5.1 状态定义

| 状态 | 含义 |
| --- | --- |
| `created` | 已生成 |
| `waiting_precondition` | 等待前置条件满足 |
| `ready` | 可开工 |
| `running` | 执行中 |
| `reported` | 已报工待确认 |
| `paused` | 暂停 |
| `locked` | 锁定 |
| `waiting_quality` | 待质量放行 |
| `done` | 已完成 |
| `closed` | 已关闭 |

## 5.2 允许迁移

| 当前状态 | 目标状态 | 触发条件 | 是否审批 |
| --- | --- | --- | --- |
| `created` | `waiting_precondition` | 工序任务生成后等待前置工序 / 物料 / 资源 | 否 |
| `waiting_precondition` | `ready` | 前置条件全部满足 | 否 |
| `ready` | `running` | 开工确认 | 否 |
| `running` | `reported` | 完成当次报工 | 否 |
| `reported` | `waiting_quality` | 本工序需要质检 | 否 |
| `reported` | `done` | 无质检要求且报工确认通过 | 否 |
| `waiting_quality` | `done` | 检验合格 | 否 |
| `running` | `paused` | 一般暂停 | 否 |
| `paused` | `running` | 恢复执行 | 否 / 视规则 |
| `running` / `reported` / `waiting_quality` | `locked` | 关键异常触发 | 否 |
| `locked` | `ready` / `running` | 解锁后恢复 | 是 |
| `done` | `closed` | 工单关闭或归档 | 否 |

## 5.3 关键约束

- 工序任务是现场最直接的执行状态对象。
- 一切自动报工、IoT 采集、报工补录、异常锁定，都应优先落到工序任务状态上。

## 六、WIP 批次状态机

## 6.1 状态定义

| 状态 | 含义 |
| --- | --- |
| `created` | WIP 批次已生成 |
| `waiting_process` | 等待进入工序 |
| `processing` | 当前工序处理中 |
| `waiting_transfer` | 工序完成等待转序 |
| `waiting_inspection` | 待检 |
| `frozen` | 冻结 / 隔离 |
| `rework_pending` | 已裁决返工，待返工执行 |
| `reworking` | 返工中 |
| `released` | 解锁放行，可继续流转 |
| `completed` | 已完成当前制造链 |
| `closed` | 已关闭归档 |

## 6.2 允许迁移

| 当前状态 | 目标状态 | 触发条件 | 是否审批 |
| --- | --- | --- | --- |
| `created` | `waiting_process` | WIP 批次建立完成 | 否 |
| `waiting_process` | `processing` | 工序接收执行 | 否 |
| `processing` | `waiting_transfer` | 当前工序完成 | 否 |
| `waiting_transfer` | `processing` | 进入下一工序 | 否 |
| `processing` / `waiting_transfer` | `waiting_inspection` | 需检验 | 否 |
| `waiting_inspection` | `released` | 检验放行通过 | 否 |
| `waiting_inspection` | `frozen` | 检验不合格 | 否 |
| `processing` / `waiting_transfer` / `waiting_inspection` | `frozen` | 质量、版本、异常锁定 | 否 |
| `frozen` | `rework_pending` | QMS 裁决为返工 | 否 |
| `rework_pending` | `reworking` | 返工启动 | 否 |
| `reworking` | `waiting_inspection` | 返工完成待复检 | 否 |
| `frozen` | `released` | 特采 / 解锁审批通过 | 是 |
| `released` | `processing` / `waiting_transfer` | 恢复现场流转 | 否 |
| `released` | `completed` | 末工序完成并放行 | 否 |
| `completed` | `closed` | 入库 / 归档完成 | 否 |

## 七、检验任务状态机

## 7.1 状态定义

| 状态 | 含义 |
| --- | --- |
| `created` | 已创建 |
| `assigned` | 已分配 |
| `inspecting` | 检验中 |
| `pending_review` | 待审核 |
| `passed` | 合格 |
| `failed` | 不合格 |
| `closed` | 已关闭 |

## 7.2 允许迁移

| 当前状态 | 目标状态 | 触发条件 | 是否审批 |
| --- | --- | --- | --- |
| `created` | `assigned` | 检验任务分派 | 否 |
| `assigned` | `inspecting` | 开始检验 | 否 |
| `inspecting` | `pending_review` | 检验录入完成 | 否 |
| `pending_review` | `passed` | 审核合格 | 否 |
| `pending_review` | `failed` | 审核判不合格 | 否 |
| `passed` / `failed` | `closed` | 后续联动动作完成 | 否 |

## 八、不合格处理单状态机

## 8.1 状态定义

| 状态 | 含义 |
| --- | --- |
| `created` | 已创建 |
| `under_review` | 评审中 |
| `rework_decided` | 已裁决返工 |
| `concession_decided` | 已裁决特采 |
| `scrap_decided` | 已裁决报废 |
| `executing` | 处置执行中 |
| `reinspection_pending` | 待复检 |
| `closed` | 已关闭 |

## 8.2 允许迁移

| 当前状态 | 目标状态 | 触发条件 | 是否审批 |
| --- | --- | --- | --- |
| `created` | `under_review` | 启动评审 | 否 |
| `under_review` | `rework_decided` | 评审结果为返工 | 否 |
| `under_review` | `concession_decided` | 评审结果为特采 | 是 |
| `under_review` | `scrap_decided` | 评审结果为报废 | 是 |
| `rework_decided` / `concession_decided` / `scrap_decided` | `executing` | 现场开始执行处置 | 否 |
| `executing` | `reinspection_pending` | 返工完成待复检 | 否 |
| `executing` | `closed` | 特采或报废处置收口完成 | 否 |
| `reinspection_pending` | `closed` | 复检关闭完成 | 否 |

## 九、版本切换任务状态机

## 9.1 状态定义

| 状态 | 含义 |
| --- | --- |
| `requested` | 已申请切换 |
| `impact_analyzing` | 影响分析中 |
| `pending_approval` | 待审批 |
| `approved` | 已批准 |
| `executing` | 切换执行中 |
| `partially_completed` | 部分完成，仍有旧版对象未收口 |
| `completed` | 全部完成 |
| `rejected` | 已驳回 |
| `closed` | 已关闭归档 |

## 9.2 允许迁移

| 当前状态 | 目标状态 | 触发条件 | 是否审批 |
| --- | --- | --- | --- |
| `requested` | `impact_analyzing` | 切换申请受理 | 否 |
| `impact_analyzing` | `pending_approval` | 影响分析完成 | 否 |
| `pending_approval` | `approved` | 审批通过 | 是 |
| `pending_approval` | `rejected` | 审批拒绝 | 是 |
| `approved` | `executing` | 现场开始切换 | 否 |
| `executing` | `partially_completed` | 新版已启用但旧版未完全收口 | 否 |
| `executing` / `partially_completed` | `completed` | 新旧版切换及收口全部完成 | 否 |
| `completed` / `rejected` | `closed` | 归档完成 | 否 |

## 十、通用锁定规则

以下事件可触发核心对象进入 `locked / frozen / paused`：

1. 缺料
2. 设备停机
3. 工艺 / 版本冲突
4. 质量不合格
5. 资质不满足
6. 审批未通过

恢复规则：

- 一般扰动可规则恢复
- 关键异常必须审批恢复

---

## 十一、状态与审批关系

| 场景 | 是否必须审批 |
| --- | --- |
| 工单强制关闭 | 是 |
| 异常解锁 | 是 |
| 特采放行 | 是 |
| 报废确认 | 是 |
| 版本切换放行 | 是 |
| 一般暂停恢复 | 否 / 可规则化 |
| 普通开工、报工、转序 | 否 |

---

## 十二、结论

后续所有页面按钮、接口动作、消息事件都应对应到本状态机上。

如果未来出现以下情况，说明设计失控了：

- 一个页面能直接把对象改成任意状态
- 两个系统都能改同一个对象状态
- 锁定对象没有明确解锁条件
- 审批通过后没有对应状态变化

只要持续守住状态机，平台的“可控执行”就能真正成立。
